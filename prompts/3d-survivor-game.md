# 3D ローグライクサバイバー — 開発指示書

## ゲーム概要
見下ろし視点の3Dサバイバーアクション（Vampire Survivors / Brotato 風）。
プレイヤーは自動攻撃する武器を持ち、四方から押し寄せる敵の大群を捌きながら生き延びる。
敵を倒すと経験値ジェムが出現し、レベルアップ時に武器や能力を選んで強くなっていく。
10分間生き延びたらクリア。

## 最重要原則
- **「動く」で終わりにしない。「触って気持ちいい」を完了条件にする。**
- すべてのプレイヤーアクション・敵の行動に視覚フィードバックをつける
- 画面上に常に何かが動いていること（パーティクル、ジェム、敵、弾）
- 数値は仮でいいから必ず入れる。空欄や TODO を残さない

## 前提条件（setup.sh で設定済み）
- URP, Input System, TextMeshPro は manifest.json に追加済み
- **URP パイプラインアセットを RunCommand で新規作成しない**。プロジェクトに既にある `UniversalRenderPipelineAsset` を `AssetDatabase.FindAssets("t:UniversalRenderPipelineAsset")` で検索して使うこと
- URP パイプラインアセットが見つからない場合: Edit > Project Settings > Graphics で手動設定を案内する（RunCommand で作ろうとしない）

## 開発フロー
1. PLANS.md を作成し、以下のフェーズを記載する
2. **1フェーズずつ進める。複数フェーズを一気に実装しない**
3. 各フェーズの全タスクを実装する
4. Camera Capture で確認し、コンセプト画像と比較する
5. PLANS.md のステータスを更新する
6. 次のフェーズへ進む

## Phase 0: コンセプトアート生成

nanobanana MCP でゲーム画面のイメージ画像を生成し、`Concepts/` フォルダに保存する。
以下の画像を生成:
1. **ゲームプレイ画面**: 見下ろし視点、ネオン発光するプレイヤー（シアン）が暗いフィールド上で赤やオレンジの敵の群れに囲まれている。画面中に光る弾丸やエフェクトが飛び交っている
2. **レベルアップ画面**: ゲームがポーズし、中央に光るカード3枚が横に並んでいる。背景は暗くぼけたゲーム画面
3. **終盤の激戦画面**: 画面を埋め尽くす大量の敵、複数の武器エフェクトが同時発動、Bloom で光があふれている

これらの画像を開発中の各フェーズで参照し、見た目の方向性を合わせる。

## Phase 1: フィールドとプレイヤー移動

### タスク
- 地面: 大きな Plane (100x100)。ダークグレーの URP Lit マテリアル、Smoothness 0.8 で床っぽい反射感
- 地面の端に壁 (高さ3の薄い Box) を配置して移動範囲を制限。壁は半透明マテリアル (Alpha 0.3)
- グリッド線: 地面の上に薄い線でグリッドを描画（ProBuilder または LineRenderer で 10m 間隔）
- プレイヤー: Capsule (高さ1.5)。明るいシアン (#00FFFF) の Emission マテリアル (Emission Intensity 3)
- プレイヤーの足元に円形の影（Projector または Plane + 影テクスチャ）
- 移動: WASD / 左スティック。New Input System の InputAction アセット使用。移動速度 8m/s
- 移動時にプレイヤーが進行方向に微かに傾く（最大15度、Lerp で滑らかに）
- 移動時に足元から小さなパーティクル（白い粒子、Lifetime 0.3s、5個/秒）
- カメラ: プレイヤーの斜め上方 (offset: 0, 18, -10)、角度60度で見下ろし。SmoothDamp で追従 (smoothTime 0.15s)
- URP Post Processing Volume:
  - Bloom: Threshold 1.0, Intensity 0.8, Scatter 0.7
  - Vignette: Intensity 0.25
  - Color Grading: Contrast +10, Saturation +15
- Directional Light: Rotation (50, -30, 0)、Intensity 1.2、色は暖色 (#FFF4E0)
- 環境光: Ambient Color をダークブルー (#0A0A2E) にして雰囲気を出す

### 完了条件
- Camera Capture でネオン感のあるフィールドにプレイヤーが立っている
- WASD で滑らかに動き、カメラが追従する
- 移動時にパーティクルが出て、体が傾く
- Bloom でプレイヤーが光って見える

## Phase 2: 敵と戦闘

### 敵の種類（5種）

| 名前 | 形状 | 色 (Emission) | HP | 速度 | 特徴 |
|---|---|---|---|---|---|
| Grunt | Sphere (0.6) | 赤 #FF3333 | 3 | 4m/s | 最も基本。まっすぐ向かってくる |
| Dasher | Capsule (0.4x0.8) | オレンジ #FF8800 | 2 | 7m/s | 高速だが HP 低い |
| Tank | Cube (1.2) | 紫 #8833FF | 12 | 2m/s | 遅いが硬い。サイズも大きい |
| Splitter | Sphere (0.8) | 緑 #33FF66 | 6 | 3.5m/s | 死亡時に小型2体に分裂 |
| Bomber | Sphere (0.7) | 黄 #FFFF33 | 4 | 5m/s | プレイヤー近接で自爆（範囲2m） |

### 敵AI
- 全敵は NavMeshAgent は使わず、`transform.position = Vector3.MoveTowards(...)` でプレイヤーに直進
- 敵同士が重なりすぎないよう、簡易的な分離力を加える（近隣の敵から逆方向に微小な力）
- Splitter の分裂: 死亡時に元の位置から左右 0.5m にオフセットして2体スポーン（サイズ 0.4、HP 2、速度 5m/s）

### スポーンシステム
- プレイヤーから半径 15〜20m のランダム位置に出現
- カメラに映らない位置（画面外）からスポーン
- スポーン頻度テーブル:

| 経過時間 | 1秒あたりスポーン数 | 出現する敵 |
|---|---|---|
| 0:00-1:00 | 0.5 | Grunt のみ |
| 1:00-2:00 | 1.0 | Grunt, Dasher |
| 2:00-4:00 | 2.0 | Grunt, Dasher, Tank |
| 4:00-6:00 | 3.0 | 全種 |
| 6:00-8:00 | 4.0 | 全種（Tank, Bomber 増加） |
| 8:00-10:00 | 5.0 | 全種（怒涛の最終フェーズ） |

- 同時に画面上に存在できる敵の最大数: 150（WebGL パフォーマンス考慮）
- 最大数を超えたら古い敵から削除

### 武器（初期武器: エナジーボルト）
- プレイヤーから最も近い敵に向かって光の弾を発射
- 発射間隔: 0.5秒
- 弾速: 15m/s
- 弾の見た目: 小さな Sphere (0.15) + Trail Renderer (長さ0.5m、シアン→透明)
- 弾のダメージ: 1
- 弾が敵に当たったら弾は消滅
- 弾の最大射程: 12m（届かなければ消滅）

### ダメージ & フィードバック（最重要）
- **敵がダメージを受けた時:**
  - 0.05秒間 白くフラッシュ（マテリアルの Emission を白 #FFFFFF に一瞬変更して戻す）
  - 弾の着弾方向に 0.2m ノックバック
  - ダメージ数値をフローティングテキストで表示（黄色、0.5秒で上に飛んでフェードアウト）
- **敵が死んだ時:**
  - パーティクルバースト: 敵の色と同じ色の粒子を 15〜20 個放出（Lifetime 0.5s、Speed 3〜5m/s）
  - 敵の Emission を一瞬 (0.1秒) 最大にしてから Destroy
  - 死亡位置に経験値ジェムをドロップ
- **プレイヤーがダメージを受けた時:**
  - 画面端を赤くフラッシュ（Vignette の色を赤、Intensity を 0.5 に一瞬上げる）
  - カメラシェイク: 0.15秒間、振幅 0.3m
  - 0.1秒のヒットストップ（Time.timeScale = 0 → 0.1秒後に戻す）
  - プレイヤーの Emission を赤に一瞬変更
  - 0.5秒の無敵時間（点滅表示）

### プレイヤーステータス
- 最大HP: 100
- 接触ダメージ: 10/回（敵に触れるとダメージ + 0.5秒無敵）
- HP 自然回復: なし（パッシブで解放可能）

### 完了条件
- 5種の敵がスポーンテーブル通りに出現する
- 弾が敵に当たり、ダメージ数値が飛び、パーティクルが出る
- プレイヤーが被弾すると画面揺れ + 赤フラッシュ + ヒットストップが発生する
- Camera Capture で「戦闘が派手で気持ちよさそう」に見える

## Phase 3: 経験値とレベルアップ

### 経験値ジェム
- 敵撃破時に死亡位置にドロップ
- 見た目: 小さな Sphere (0.2) + Emission + 上下に浮遊する Bobbing アニメーション (振幅 0.15m、周期 1秒)
- 色: 水色 #66FFFF (Emission Intensity 5)
- 吸い寄せ: プレイヤーから半径 2m 以内に入ると加速してプレイヤーに向かう（初速 5m/s → 加速）
- 取得時: 小さなパーティクル + 「キラン」的な光の放射

### レベルアップ
- 必要経験値: Level * 5（Lv1→5, Lv2→10, Lv3→15, ...）
- レベルアップ演出:
  - Time.timeScale = 0 にしてゲームをポーズ
  - プレイヤーから光の輪が広がるエフェクト
  - 画面中央に選択 UI を表示

### レベルアップ選択 UI
- 3択をカードとして表示（横に3枚並べる）
- カードの見た目: 角丸風の Panel、上にアイコン（色付き Sphere/Cube）、下にテキスト
- カードにマウスホバーで拡大 (1.1倍) + 枠線が光る
- クリックで選択 → カードが消えて TimeScale を 1 に戻す

### 武器（6種）

| 名前 | 説明 | レベルアップ効果 |
|---|---|---|
| エナジーボルト | 最寄りの敵に弾を発射（初期装備） | 発射数+1（最大5方向に同時発射） |
| フレイムオーラ | プレイヤー周囲にダメージ範囲 (半径 2m) | 範囲拡大 +0.5m、ダメージ+2 |
| フロストノヴァ | 3秒ごとに周囲の敵を減速 (50%) + ダメージ | クールダウン -0.5秒、範囲+1m |
| ライトニングチェイン | ランダムな敵に雷を落とし、近くの敵に連鎖 (最大3体) | 連鎖数+1、ダメージ+1 |
| オービタルブレード | プレイヤーの周囲を回転する刃 (半径 2.5m) | 刃の数+1（最大4枚） |
| ミサイルスウォーム | 2秒ごとにランダム方向にミサイル3発。着弾で爆発 (範囲1.5m) | ミサイル+2発、爆発範囲+0.3m |

- 各武器には専用のパーティクルエフェクトをつける
- エフェクトの色は武器ごとに統一（ボルト=シアン、フレイム=オレンジ、フロスト=水色、ライトニング=黄、ブレード=白、ミサイル=赤）

### パッシブ能力（4種）

| 名前 | 効果 | レベルアップ |
|---|---|---|
| スピードブースト | 移動速度 +15% | +15% ずつ加算 |
| マグネット | ジェム吸い寄せ範囲 +50% | +50% ずつ加算 |
| リジェネ | 毎秒 HP +1 回復 | +1/秒 ずつ加算 |
| アーマー | 被ダメージ -5 (最低1) | -5 ずつ加算 |

### 完了条件
- ジェムが浮遊してキラキラしていて、吸い込まれる動きが気持ちいい
- レベルアップ UI がポーズ中に表示され、ホバーでカードが反応する
- 武器を選ぶと即座にゲームに反映される（2つ目の武器を取ると2種が同時に発動）
- Camera Capture で武器エフェクトが画面を賑やかに彩っている

## Phase 4: ゲームフィール強化（ポリッシュ専用フェーズ）

このフェーズは新機能追加ではなく、既存要素の手触りを良くすることに全力を注ぐ。

### 追加すべき演出
- **キルコンボ**: 1秒以内に連続キルすると画面右にコンボカウンター表示 (x2, x3...)
  - コンボ数に応じてフォントサイズが大きくなる
  - 10コンボ以上で画面全体が一瞬白くフラッシュ
- **キルストリーク演出**: 50キルごとに「50 KILLS!」のテキストが画面中央に大きく表示されて消える
- **画面上の敵数が 50 を超えたら**:
  - BGM がない代わりに、環境音的にブーン（低い振動音）を鳴らす…は音声系は不要。代わりに Bloom Intensity を 1.0 → 1.5 に徐々に上げて緊迫感を演出
- **プレイヤーの移動トレイル**: Trail Renderer (幅 0.3m、シアン→透明、長さ 1m)
- **敵の大群が来ている方向**: 画面端に赤い矢印インジケーターを表示（画面外の敵集団の方向）
- **スローモーション**: レベルアップ直後の 0.5 秒間、Time.timeScale = 0.3 にして新武器の威力を実感させる
- **ジェム大量取得時**: 一度に 5 個以上取ると取得パーティクルが大きくなる

### パフォーマンス最適化（WebGL 向け）
- 敵のオブジェクトプーリング: Instantiate/Destroy を毎回呼ばず、プールから取得・返却
- 弾のオブジェクトプーリング: 同上
- パーティクルは ParticleSystem の maxParticles を制限（システム全体で 500 以下）
- 敵が 100 体を超えたら LOD 的に遠くの敵の更新頻度を落とす（2フレームに1回）

### 完了条件
- 3分間プレイして「気持ちいい」「もう1回やりたい」と思えるレベル
- コンボカウンターが戦闘のリズムを生んでいる
- 画面が常に賑やかで、退屈な瞬間がない
- Camera Capture でエフェクトまみれの戦闘シーンが確認できる

## Phase 5: UI とゲームフロー

### タイトル画面
- ゲーム名を大きく表示（TextMeshPro、白、Outline + Glow）
- 背景: ゲームフィールドをぼかして表示（カメラの Depth of Field）、または暗い背景に浮遊するパーティクル
- 「CLICK TO START」テキストが点滅（Alpha を Sin でアニメーション）
- クリックでゲームシーンへ遷移

### HUD（ゲーム中常時表示）
- 左上: HP バー（緑→黄→赤のグラデーション、現在HP / 最大HP テキスト）
- 左上 HP 下: 経験値バー（シアン、現在の経験値がバーで表示）
- 右上: 経過時間 (MM:SS)、撃破数
- 右側: キルコンボ表示領域
- 下部中央: 現在装備中の武器アイコン一覧（最大6スロット）

### ゲームオーバー画面
- 「GAME OVER」大きく表示 + 画面暗転
- 統計表示: 生存時間、撃破数、最大コンボ、到達レベル
- 「RETRY」ボタン（ホバーで光る）
- 「TITLE」ボタン

### クリア画面（10分生存時）
- 「SURVIVED!」大きく表示 + 画面全体が白くフラッシュ→通常に戻る
- 統計表示（ゲームオーバーと同じ）
- 「RETRY」「TITLE」ボタン

### ポーズ
- Escape キーでポーズ切り替え
- 半透明の黒オーバーレイ + 「PAUSED」テキスト
- 「RESUME」「QUIT TO TITLE」ボタン

### 完了条件
- タイトル → ゲーム → ゲームオーバー → リトライ → ゲーム → ポーズ → タイトル の全フローが動作
- HUD がゲームプレイを邪魔せず、必要な情報が一目でわかる
- Camera Capture で UI が整っている

## 技術制約
- 外部アセット不使用（プリミティブ + ParticleSystem + URP マテリアルのみ）
- WebGL 互換: `System.IO`, `Thread`, `Task.Run` は使用不可
- New Input System 必須（`UnityEngine.InputSystem`）
- オブジェクトプーリングで GC を抑制
- 同時敵数上限 150、パーティクル上限 500
